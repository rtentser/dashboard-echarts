# Типы диаграмм и компонентов

Документ описывает актуальный формат конфигурации диаграмм и layout-компонентов.

Конфигурация страницы находится в `src/config/chartConfig.json`.

Данные для диаграмм, использующих `dataSource`, берутся из `mock.json` или из API, в зависимости от переменных окружения.

---

## Общая структура страницы

Корневая конфигурация содержит массив `layout`:

```json
{
  "layout": [
    {
      "type": "chart",
      "config": {
        "title": "Текущий остаток денежных средств",
        "type": "single-value",
        "value": 19827961,
        "unit": "₽"
      }
    }
  ]
}
```

Элементы `layout` могут быть:

- `chart`
- `tabs-block`
- `group-horizontal`
- `group-vertical`

---

## Источник данных `dataSource`

Для диаграмм, которые строятся по данным из `mock.json` или API, используется поле `dataSource`.

Пути к данным задаются через dot-notation:

```json
{
  "xField": "общее.месяц",
  "yField": "ДДС.Остаток денег на конец месяца"
}
```

Пример:

```json
{
  "title": "Остаток денег",
  "type": "bar-chart",
  "dataSource": {
    "xField": "общее.месяц",
    "yField": "ДДС.Остаток денег на конец месяца"
  }
}
```

Такой путь:

```text
ДДС.Остаток денег на конец месяца
```

означает обращение к данным:

```ts
data["ДДС"]["Остаток денег на конец месяца"]
```

## Диаграммы

### 1. `single-value` — одно значение

Простое отображение одного значения.

```json
{
  "title": "Текущий остаток денежных средств",
  "type": "single-value",
  "value": 19827961,
  "unit": "₽"
}
```

Поля:

| Поле | Обязательно | Описание |
|---|---:|---|
| `title` | да | Название метрики |
| `type` | да | Всегда `single-value` |
| `value` | да | Числовое или строковое значение |
| `unit` | нет | Единица измерения |

---

### 2. `summary` — сводная метрика

Показывает текущее значение и изменение относительно предыдущего значения.

#### Вариант с прямыми значениями

```json
{
  "title": "Остаток денег",
  "type": "summary",
  "currentValue": 125000,
  "previousValue": 100000,
  "unit": "₽"
}
```

#### Вариант через `dataSource`

```json
{
  "title": "Остаток денег",
  "type": "summary",
  "dataSource": {
    "valueField": "ДДС.Остаток денег на конец месяца"
  },
  "unit": "₽"
}
```

Поля:

| Поле | Обязательно | Описание |
|---|---:|---|
| `title` | да | Название метрики |
| `type` | да | Всегда `summary` |
| `currentValue` | да, если нет `dataSource` | Текущее значение |
| `previousValue` | да, если нет `dataSource` | Предыдущее значение |
| `dataSource.valueField` | да, если используется `dataSource` | Путь к массиву значений |
| `unit` | нет | Единица измерения |

---

### 3. `bar-chart` — столбчатая диаграмма

#### Один ряд столбцов

```json
{
  "title": "Поступления по ОД",
  "type": "bar-chart",
  "dataSource": {
    "xField": "общее.месяц",
    "yField": "ДДС.Поступления по ОД"
  }
}
```

#### Несколько рядов столбцов

Для нескольких столбцов используется `yFields`.

```json
{
  "title": "Поступления и выбытия",
  "type": "bar-chart",
  "dataSource": {
    "xField": "общее.месяц",
    "yFields": [
      {
        "name": "Поступления по ОД",
        "field": "ДДС.Поступления по ОД"
      },
      {
        "name": "Выбытия по ОД",
        "field": "ДДС.Выбытия по ОД"
      }
    ]
  }
}
```

Поля:

| Поле | Обязательно | Описание |
|---|---:|---|
| `title` | нет | Заголовок диаграммы |
| `type` | да | Всегда `bar-chart` |
| `dataSource.xField` | да | Путь к данным для оси X |
| `dataSource.yField` | да, если одна серия | Путь к значениям одной серии |
| `dataSource.yFields` | да, если несколько серий | Массив серий `{ name, field }` |

### 5. `combo-chart` — комбинированная диаграмма

Комбинированная диаграмма отображает столбцы и линии на одном графике.

Актуальный формат для `combo-chart`:

```json
{
  "title": "Остаток денег на конец месяца",
  "type": "combo-chart",
  "dataSource": {
    "xField": "общее.месяц",
    "barFields": [
      {
        "name": "Остаток денег",
        "field": "ДДС.Остаток денег на конец месяца"
      }
    ],
    "lineFields": [
      {
        "name": "Комфортные коридоры",
        "field": "ДДС.Комфортные коридоры по остаткам денег"
      }
    ]
  }
}
```

#### Несколько столбцов и несколько линий

```json
{
  "title": "Остаток денег и коридоры",
  "type": "combo-chart",
  "dataSource": {
    "xField": "общее.месяц",
    "barFields": [
      {
        "name": "Остаток денег",
        "field": "ДДС.Остаток денег на конец месяца"
      },
      {
        "name": "Поступления по ОД",
        "field": "ДДС.Поступления по ОД"
      }
    ],
    "lineFields": [
      {
        "name": "Комфортные коридоры",
        "field": "ДДС.Комфортные коридоры по остаткам денег"
      },
      {
        "name": "Общий денежный поток",
        "field": "ДДС.Общий денежный поток"
      }
    ]
  }
}
```

Поля:

| Поле | Обязательно | Описание |
|---|---:|---|
| `title` | нет | Заголовок диаграммы |
| `type` | да | Всегда `combo-chart` |
| `dataSource.xField` | да | Путь к данным для оси X |
| `dataSource.barFields` | нет | Массив столбчатых серий `{ name, field }` |
| `dataSource.lineFields` | нет | Массив линейных серий `{ name, field }` |

Важно: для `combo-chart` нужно использовать именно `xField`, `barFields` и `lineFields`.

Неактуальный формат ниже использовать не нужно:

```json
{
  "dataSource": {
    "x": "общее.месяц",
    "bar": "ДДС.Остаток денег на конец месяца",
    "line": "ДДС.Комфортные коридоры по остаткам денег"
  }
}
```

### 6. `area-chart` — диаграмма с областью

Линейная диаграмма с закрашенной областью под линией.

`area-chart` использует `dataSource` так же, как одиночная `line-chart`: ось X берётся из `xField`, значения линии — из `yField`.

```json
{
  "title": "Накопление капитала",
  "type": "area-chart",
  "dataSource": {
    "xField": "общее.месяц",
    "yFields": [
      {
        "name": "Остаток денег",
        "field": "ДДС.Остаток денег на конец месяца"
      }
    ]
  }
}
```

Пример данных:

```json
{
  "общее": {
    "месяц": ["Янв", "Фев", "Мар"]
  },
  "ДДС": {
    "Остаток денег на конец месяца": [100000, 125000, 140000]
  }
}
```

При использовании внутри `layout` диаграмму нужно оборачивать в компонент `chart`:

```json
{
  "type": "chart",
  "config": {
    "title": "Накопление капитала",
    "type": "area-chart",
    "dataSource": {
      "xField": "общее.месяц",
      "yField": "ДДС.Остаток денег на конец месяца"
    }
  }
}
```

Поля:

| Поле | Обязательно | Описание |
|---|---:|---|
| `title` | нет | Заголовок диаграммы |
| `type` | да | Всегда `area-chart` |
| `dataSource.xField` | да | Путь к массиву подписей для оси X |
| `dataSource.yFields` | да | Массив серий `{ name, field }`. Даже для одной области используется массив с одним элементом |

## Layout-компоненты

### `chart` — компонент диаграммы

Оборачивает одну диаграмму.

```json
{
  "type": "chart",
  "config": {
    "title": "Поступления по ОД",
    "type": "bar-chart",
    "dataSource": {
      "xField": "общее.месяц",
      "yField": "ДДС.Поступления по ОД"
    }
  }
}
```

Поля:

| Поле | Обязательно | Описание |
|---|---:|---|
| `type` | да | Всегда `chart` |
| `config` | да | Конфигурация диаграммы |

---

### `group-horizontal` — горизонтальная группа

Размещает компоненты горизонтально в одной строке.

```json
{
  "type": "group-horizontal",
  "title": "Ключевые метрики",
  "items": [
    {
      "type": "chart",
      "config": {
        "title": "Метрика 1",
        "type": "single-value",
        "value": 100
      }
    },
    {
      "type": "chart",
      "config": {
        "title": "Метрика 2",
        "type": "single-value",
        "value": 200
      }
    }
  ]
}
```

Поля:

| Поле | Обязательно | Описание |
|---|---:|---|
| `type` | да | Всегда `group-horizontal` |
| `title` | нет | Заголовок группы |
| `items` | да | Массив вложенных компонентов |

---

```json

        }

```

Поля:

| Поле | Обязательно | Описание |
|---|---:|---|
| `type` | да | Всегда `group-vertical` |
| `title` | да | Заголовок группы |
| `items` | да | Массив вложенных компонентов |

### `tabs-block` — блок с вкладками

Группирует компоненты по вкладкам.

Актуальный формат использует `items` внутри каждой вкладки.

```json
{
  "type": "tabs-block",
  "title": "Финансовые показатели",
  "tabs": [
    {
      "name": "Денежный поток",
      "items": [
        {
          "type": "chart",
          "config": {
            "title": "Поступления по ОД",
            "type": "bar-chart",
            "dataSource": {
              "xField": "общее.месяц",
              "yField": "ДДС.Поступления по ОД"
            }
          }
        }
      ]
    },
    {
      "name": "P&L",
      "items": [
        {
          "type": "chart",
          "config": {
            "title": "Зарплата",
            "type": "bar-chart",
            "dataSource": {
              "xField": "общее.месяц",
              "yField": "ДДС.Зарплата"
            }
          }
        }
      ]
    }
  ]
}
```

Поля:

| Поле | Обязательно | Описание |
|---|---:|---|
| `type` | да | Всегда `tabs-block` |
| `title` | нет | Заголовок блока вкладок |
| `tabs` | да | Массив вкладок |
| `tabs[].name` | да | Название вкладки |
| `tabs[].items` | да | Массив компонентов внутри вкладки |

Неактуальный формат с `charts` внутри вкладок использовать не нужно:

```json
{
  "tabs": [
    {
      "name": "Продажи",
      "charts": []
    }
  ]
}
```

## Частые ошибки

### `Could not find fields: undefined, undefined`

Обычно означает, что диаграмма ожидает `xField` и `yField`, но в конфиге указаны другие поля.

Для `combo-chart` нельзя использовать такой формат:

```json
{
  "dataSource": {
    "x": "общее.месяц",
    "bar": "ДДС.Остаток денег на конец месяца",
    "line": "ДДС.Комфортные коридоры по остаткам денег"
  }
}
```

Нужно использовать такой формат:

```json
{
  "dataSource": {
    "xField": "общее.месяц",
    "barFields": [
      {
        "name": "Остаток денег",
        "field": "ДДС.Остаток денег на конец месяца"
      }
    ],
    "lineFields": [
      {
        "name": "Комфортные коридоры",
        "field": "ДДС.Комфортные коридоры по остаткам денег"
      }
    ]
  }
}
```

### `tabs[].charts` не работает

В актуальной структуре вкладки используют `items`, а не `charts`.

Неправильно:

```json
{
  "name": "Денежный поток",
  "charts": []
}
```

Правильно:

```json
{
  "name": "Денежный поток",
  "items": []
}
```

## Источник данных

По умолчанию данные берутся из `mock.json`.

При использовании mock-данных:

```env
VITE_USE_MOCK=true
```

При использовании API:

```env
VITE_USE_MOCK=false
VITE_API_URL=https://example.com/api/data
```

Все поля в `dataSource` должны соответствовать структуре данных, которые возвращает `mock.json` или API.
